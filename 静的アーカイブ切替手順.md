# 静的アーカイブモード 切替手順

イベント（2026年9月）の投稿収集・モデレーションが完了したあと、Supabaseを止めて静的JSONだけでサイトを運用するための手順です。

## 1. 前提

- `/admin` でのモデレーション（承認 / 非表示）が完了していること。
- 公開してよい手紙だけが `approved = true`, `hidden = false`, `visibility = 'public'` になっていることを確認する。

## 2. 焼き込み（ローカル）

1. ローカルの `.env.local` に Supabase の接続情報が入っていることを確認する。
   - `NEXT_PUBLIC_SUPABASE_URL`
   - `SUPABASE_SERVICE_ROLE_KEY`
2. プロジェクトルート（`natsu-time-capsule/`）で以下を実行する。

   ```bash
   npm run bake
   ```

3. `lib/letters-data.json` が公開対象の手紙で更新される。メール・非公開・未承認・非表示の手紙は含まれない。

## 3. 確認

- `lib/letters-data.json` の件数が想定通りか目視確認する（`approved`件数と一致するはず）。
- 中身に個人情報や非公開扱いの手紙が混ざっていないか確認する。

## 4. コミット・push

```bash
git add -A
git commit -m "静的アーカイブ: 手紙データを焼き込み"
git push
```

## 5. Vercel環境変数の追加

Vercelのプロジェクト設定 → Environment Variables に以下を追加する。

- `LETTERS_SOURCE=static`
- `NEXT_PUBLIC_SUBMISSIONS_OPEN=false`

## 6. 再デプロイ

Vercelで再デプロイする（環境変数追加後は自動、または手動でRedeployを実行）。

## 7. 動作確認

- トップページの投稿フォームが消え、「受付は終了しました」の表示になっていること。
- 開封カウントダウンのpanelは引き続き表示されていること。
- 公開日（`REVEAL_ISO`）より前は `/api/letters` が引き続きロックされたままであること（＝公開日まで手紙は表に出ない）。管理者ログイン中はプレビュー表示できる点は従来通り。
- 公開日以降、`/letters` ページで `lib/letters-data.json` の内容が表示されること。

## 8. （任意）GitHubリポジトリをPrivateに切替

`lib/letters-data.json` に手紙本文が入るため、公開日より前にリポジトリが誰でも見える状態だと手紙が早期に読めてしまう。必要であればPrivateに切り替える。

## 9. Supabaseの停止・削除

上記が完了し、静的モードで正しく動作していることを確認したら、Supabaseプロジェクトを停止・削除してよい。

- keep-alive処理・月次CSVバックアップは不要になる。
- `SUPABASE_*` 系の環境変数は以後使われなくなる。
- Supabaseに依存する管理画面 `/admin`（投稿の承認・非表示切り替え、一覧取得など）は、Supabase停止後は動作しなくなる。モデレーションはこの時点で完了している前提のため問題ない。
